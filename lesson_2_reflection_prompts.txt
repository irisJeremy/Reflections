How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area is a space to work on documents before saving newer changes for them in the repository. The directory is simply a visual tool for finding the most recent version of those documents. The value in this is that you have a backup system to your work in the event that you make document mistakes. 

How can you use the staging area to make sure you have one commit per logical change?
An individual can utililze "git add" to push one logical change to the staging area to be committed. 

How do the diagrams help you visualize the branch structure?

Diagrams allow you to see what you need to address if you plan to combine branches.
