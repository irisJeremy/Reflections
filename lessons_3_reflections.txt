When would you want to use a remote repository rather than keeping all your work
local?

A remote repository can be accessed by others, so you may prefer to use remote repo's when you are working in a group. 

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

One may not want to pull every branch from a repo, and so the changes that are pulled manually can be easier managed. It may also be the case that there will be conflict between repositories that you have not finished resolving yet.
